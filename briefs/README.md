# Briefs

This folder contains reusable knowledge documents about various topics learned during development. Briefs are project-independent and can be referenced across different projects.

## What are Briefs?

Briefs are short, focused documents that capture specific knowledge or patterns:
- **Project-independent**: Not tied to any specific project
- **Reusable**: Referenced across different projects
- **Self-contained**: Include practical examples and clear explanations

## Frontmatter Format

Every brief includes YAML frontmatter with metadata:

```yaml
---
title: Brief Title
description: Brief description of what this covers
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags:
  - tag1
  - tag2
category: Category Name (Optional)
author: Author/source (Optional)
references:
  - https://example.com (Optional)
---
```

## Creating a Brief

1. Create a new markdown file in this `briefs/` folder
2. Add proper frontmatter at the top
3. Include practical examples
4. Use clear structure with headings and code blocks
5. Update the `updated` date to today
